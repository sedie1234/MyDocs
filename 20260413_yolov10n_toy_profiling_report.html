⚠ 이 보고서는 "toy" 버전 시뮬레이션 기준입니다.
이 시뮬레이션은 실제 ONNX 그래프가 아닌, 작업 초반에 수작업으로 만든 합성 .ihnn 파일 (83개 Conv, 각각 1 tile, single pcore) 을 돌린 것입니다. 실제 yolov10n 의 Concat/Split/Add/Resize/ MaxPool/Sigmoid/Gemm 등은 포함돼 있지 않습니다.
ONNX 실제 그래프 기반 결과는 20260413_yolov10n_profiling_report.html 을 보세요.

병목 분석

초기 Layer 병목

  • Layer 0 (model.0): 1,228,800 compute cycles = 1.54 ms
  • Layer 1 (model.1): 1,638,400 compute cycles = 2.05 ms
  • 두 layer 합 = 전체의 8.25%
  • 원인: 640×640 입력에서의 3×3 Conv, 채널 증가

Single Core 제약

  • Pcore 0 만 사용, 나머지 11 pcore + 12 acore idle
  • Pcore 0 max ref_count entries: 167 (모든 write lock 이 pcore 0 에 집중)
  • 12-core 이상적 분산 시 이론적 3.6 ms → 실측 43.6 ms

DMA Overlap 부재

  • Compute + DMA = 34.88M cycles = wall time 과 동일
  • 즉 DMA 와 compute 가 직렬 실행 되고 있음
  • Double-buffering 활성화 시 추가 20~30% 단축 여지

Weight DMA Overhead

  • 각 LAYER_START 에서 weight 를 SM→UB 로 전송
  • Weight size: 112~36,868 cycles 범위
  • 대형 weight (147,456 B 기준) 은 36,868 cycles 소요
