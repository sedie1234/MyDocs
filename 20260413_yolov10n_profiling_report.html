남은 과제

1. SM 121.4% 초과 (~900 KB)

  • Weights metadata 2.29 MB 고정 + FM peak 2.64 MB > 4 MB 한도
  • 원인: model.0/1 시점 graph input (1.17 MB) + 초기 FM 동시 live
  • 해결책 후보:
    • Graph input tile-streaming (runtime 변경)
    • Chain 역순 tile 처리 (input 위에 output 덮어쓰기)
    • Weight 를 SM 밖으로 (agenda 제외)

2. Critical path 48 ms (pcore 11 집중)

  • Simulator 가 pcore 를 index 순 직렬 실행 → 진짜 병렬이 아님
  • 실제 HW 에서 critical path ≈ max(per-core work) 로 떨어질 가능성
  • Chain 추가 확장(depthwise 포함)으로 뒷 pcore 부담 분산 가능

3. Op fusion ON 비교

  • Fusion OFF 기준 현재 수치
  • Fusion ON 시 ERRORS 51, Queue 23% 로 추가 최적화 (단 semantic 변화)
  • 두 모드의 cycle 수 비교가 남음

4. Layer-level breakdown

  • 3173 event log 가 있으므로 op 별 cycle 분석 가능
  • 어떤 layer 가 critical path 인지, compute vs DMA bound 분류
