# Skill: 모델 요구사항 분석

> **사용자**: HW 설계자, 프로젝트 매니저  
> **상황**: 특정 모델을 NPU에서 실행하려면 하드웨어가 최소 어느 정도 스펙이어야 하는지 알고 싶을 때  
> **사용법**: 모델명 또는 모델 스펙을 입력

---

## 프롬프트 1: 기존 모델 요구사항

```
【YOLOv10n】을 Infetron-V2에서 실행하기 위한 최소 하드웨어 요구사항을 알려줘.

다음 항목별로:
- SM: 최소 몇 MiB 필요?
- UB: 최소 몇 KiB 필요? (가장 큰 weight layer 기준)
- Q: 최소 몇 records 필요? (최적 타일 기준)
- 코어: 최소 몇 개?

각 값의 근거(어떤 layer가 병목인지)도 함께 알려줘.
```

---

## 프롬프트 2: 새 모델 분석 요청

```
다음 모델의 HW 요구사항을 분석해줘.

## 모델 정보
- 이름: 【모델명】
- Conv layers: 【N개】
- 총 weight: 【X MB】
- 최대 단일 weight: 【Y KB】 (어떤 layer인지)
- 입력 크기: 【H×W×C, dtype】
- 최대 채널 폭: 【Z】

※ 정확한 layer 정보가 없으면 위 수치만으로 대략적인 feasibility를 추정해줘.
```

---

## 프롬프트 3: "이 모델이 돌아가나?"

```
현재 HW 스펙(Q=512, UB=256KiB, SM=4MiB, 12코어)에서
【모델명 또는 스펙: 총 weight X MB, 최대 단일 weight Y KB, 입력 H×W×C】
이 돌아가는지 빠르게 판단해줘.

돌아가면: 여유가 얼마나 되는지
안 돌아가면: 어떤 자원이 부족한지, 최소 얼마로 늘려야 하는지
```

---

## 프롬프트 4: 지원 가능 모델 범위

```
현재 HW 스펙에서 실행 가능한 모델의 upper bound를 알려줘.

- 총 weight 최대 몇 MB까지?
- 단일 layer weight 최대 몇 KB까지?
- 최대 채널 폭?
- 최대 입력 해상도?

각 한계를 결정하는 자원(Q/UB/SM)과 계산 과정을 보여줘.
```
