# Skill: 컴파일러 지원 가능 여부 판단

> **사용자**: HW 설계자, ISA 설계자, 런타임 개발자  
> **상황**: 새로운 하드웨어/ISA 기능을 설계할 때 컴파일러가 대응 가능한지 미리 확인하고 싶을 때  
> **사용법**: 아래 프롬프트의 【 】 부분을 자신의 가정으로 채워서 입력

---

## 프롬프트

```
다음 하드웨어 가정에 대해 컴파일러 지원 가능 여부를 판단해줘.
"컴파일러 지원 가능 여부 판단 지식 문서"의 출력 포맷(§5)을 따를 것.

## 가정
【여기에 HW/ISA 변경 가정을 자유롭게 기술】

예시:
- "UB에 weight를 적재하지 않고 SM에서 직접 읽는 방식(weight streaming)을 ISA로 지원하려 한다"
- "fp16 연산을 HW에서 지원하면 컴파일러가 활용할 수 있나?"
- "코어를 16개로 늘리고 4×4 mesh 토폴로지로 바꾸려 한다"
- "Q record 크기를 512B로 늘리려 한다"

## 특히 알고 싶은 점
【예: 컴파일러 코드 수정이 필요한지, 어느 파일을 바꿔야 하는지, 기간이 얼마나 걸릴지】
```

---

## 간단 버전

```
【Weight streaming】을 ISA로 지원하면 컴파일러에 어떤 영향이 있나?
판정 등급(A~D)과 수정이 필요한 코드 목록을 알려줘.
```

---

## 복수 가정 동시 질문

```
다음 3가지를 동시에 적용하면 컴파일러 대응이 가능한가?
1. 【UB 512 KiB】
2. 【Weight streaming】
3. 【코어 16개】

각각 개별 판정 + 조합 판정을 해줘.
```

---

## 예상 응답 형식

| 항목 | 내용 |
|:---|:---|
| **판정** | A / B / C / D |
| **영향 영역** | 영향받는 Pass/단계 |
| **코드 변경 규모** | config만 / Pass 수정 / 파이프라인 변경 |
| **선행 조건** | ISA 확정 등 |
| **Feasibility 영향** | Q/UB/SM 수치 변화 |
